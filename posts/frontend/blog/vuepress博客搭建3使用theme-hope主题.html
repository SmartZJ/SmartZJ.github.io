                    # vuepress博客搭建3使用theme-hope主题

                    前面两章完成了博客基础搭建和默认主题的配置，看着白白的博客总觉得它还缺了点什么。这个博客太简约太像文档了，于是乎我得想个办法让它花里胡哨起来。所以今天要给它加上一个theme-hope主题，让博客完成华丽变身。

                    theme-hope官方文档open in new window

                    # 安装主题

                    通过粗略的研读官方文档我没有看到从原项目升级到主题的办法所以咱们就新建一个工程吧，反正博客网站重要的是md文件。

                    # 环境要求

                    • 环境: LTS 版本 的 Node.js（18、20）
                    • 包管理器: npm >= 8、yarn >= 2 或 pnpm >= 7
                    • 使用 Vue3 的 VuePress2 项目

                    # 创建项目

                    在想要创建项目的文件夹中呼出命令行

                    npm
                    npm init vuepress-theme-hope my-docs
                    pnpm
                    pnpm create vuepress-theme-hope my-docs
                    yarn
                    yarn create vuepress-theme-hope my-docs

                    提示

                    命令行最后一个参数是项目文件夹命可自定义

                    根据提示完成配置

                    用vscode打开项目文件夹直接运行项目。

                    # 自定义项目

                    这次我将以我的博客为蓝本讲述我是如何自定义我的项目的，可能不全面但可以快速上手这个主题。

                    # logo修改

                    开始修改项目之前我们还是一起来看看hope主题项目的结构有什么不同。

                    基于对vuepress的项目我们来理解一下这个项目结构，src相当于vuepress中的docs，也就是存放博客文章的文件夹。

                    .vurpress依旧是最重要的文件夹，项目配置文件config.ts在其中，但不同的是我看到了之前没有见过的theme.ts、navbar.ts、siderbar.ts三个配置文件，这其实是作者为了保证配置导航栏、侧边栏、主题的便捷性，利用es 6的新特性将这些组件的配置参数抽离成了各自单独配置文件。

                    以下是config.ts的内容：

                    import { defineUserConfig } from "vuepress";
import theme from "./theme.js";

export default defineUserConfig({
  base: "/",
  lang: "zh-CN",
  title: "博客演示",
  description: "vuepress-theme-hope 的博客演示",
  theme,
  // Enable it with pwa
  // shouldPrefetch: false,
});

                    这个眼熟的文件就是vuepress配置文件，修改title即可修改网站的名称。

                    可以看到第二行代码对主题配置文件的引入，所以我们对项目配置的重点就应该放到theme.ts这个文件中。

                    以下是theme.ts的局部内容,为了方便查看我们需要修改的参数，后面都将默认只展示关键代码。

                    import { hopeTheme } from "vuepress-theme-hope";
import navbar from "./navbar.js";
import sidebar from "./sidebar.js";
export default hopeTheme({
  hostname: "https://mister-hope.github.io",
  author: {
    name: "Mr.Hope",
    url: "https://mister-hope.com",
  },
  iconAssets: "fontawesome-with-brands",
  logo: "/logo.svg",
  repo: "vuepress-theme-hope/vuepress-theme-hope",
  // navbar
  navbar,
  // sidebar
  sidebar,
  footer: "默认页脚",
});

                    # 参数说明

                    • author ： 作者信息配置，包含两个参数name、url,name展示作者姓名，url保存一个网址。该主题会在文章的简介左下角展示一个作者名称，配置这个参数可以让作者名称的点击事件触发网页跳转，跳转到配置的url网页中。
                    • iconAssets: 配置icon资源的引入。
                    • logo： 配置网站左上角的logo。
                    • repo: 项目仓库的 URL，详细配置可以去[vupress配置](配置 | VuePress (vuejs.org)open in new window)中查询。
                    • navbar： 导航配置。
                    • sidebar：侧边栏配置。

                    到此我们就看到了这个小结的目标参数：logo,修改logo的参数为我们自己的logo。

                    # 导航栏配置

                    打开navbar.ts可以看到如下内容：（精简后的代码）

                    export default navbar([
  "/",
  "/demo/",
  {
    text: "博文",
    icon: "pen-to-square",
    prefix: "/posts/",
    children: [
      {
        text: "苹果",
        icon: "pen-to-square",
        prefix: "apple/",
        children: [
          { text: "苹果1", icon: "pen-to-square", link: "1" },
          ......
          "4",
        ],
      },
      ......
      { text: "樱桃", icon: "pen-to-square", link: "cherry" },
      ......
      "strawberry",
    ],
  },
  {
    text: "V2 文档",
    icon: "book",
    link: "https://theme-hope.vuejs.press/zh/",
  },
]);

                    界面的呈现是这样：

                    关于prefix

                    它是前缀添加属性，就是对于children中配置的路径，都会在前面都会追加prefix里的参数。

                    如：”苹果1“的link是”1“，”苹果1“的父节点"苹果"perfix属性为"apple/"，”苹果“的父节点”博文“perfix属性为"/posts/"，所以”苹果1“在运行时的路径就是”/posts/apple/1“

                    由文件可见配置方式一共有三种：

                    1. 路径配置，例如："/"。 这样配置的导航子项会自动寻找对应路径下的README.md,并且读取md文件中的frontmatter关于标题和icon配置作为具体呈现。
                    2. 文件名配置，例如："4"。 这样配置的导航子项会自动寻找对应路径下的4.md文件,并且读取md文件中的frontmatter关于标题和icon配置作为具体呈现。
                    3. navbarItem配置,这个和vuepress的默认主题配置一样就不多说了。

                    注意路径

                    一定要在路径最后加 / ,不然会出现多次点击跳转，路径不停拼接导致404的情况。

                    # 主题模式切换设置

                    hope主题默认提供了三种模式：深色、浅色、自动，但是点到自动选项时没有任何效果产生，我感觉体验不好，所以准备把它改成手动明暗切换。

                    修改主题切换需要配置 darkmode这个属性一共有一下几种参数可填写。

                    • "switch": 在深色模式，浅色模式和自动之间切换
                    • "toggle": 在深色模式和浅色模式之间切换
                    • "auto": 自动根据用户设备主题或当前时间决定是否应用深色模式
                    • "enable": 强制深色模式
                    • "disable": 禁用深色模式

                    我只需要深浅两种模式，所以需要在theme.ts中做出如下修改：

                    export default hopeTheme({
    ...
    darkmode: "toggle",
    ...
});

                    # 主站图标和口号标语修改

                    这个两个配置其实算是页面配置，是在src文件夹下的README.md文件的frontmatter配置中修改。

                    主页frontmatter文件内容如下：

                    ---
home: true
layout: BlogHome
icon: home
title: 博客主页
heroImage: /logo.svg
heroText: 你博客的名称
tagline: 你可以在这里放置你的口号与标语
---

                    这个文件内容对应页面呈现一看就能知道怎么改，但为了篇幅我还是啰嗦一下。

                    • icon和title就是刚刚在讲导航配置时导航栏会读取的属性。
                    • heroImage就是主站图标，可以换上你自己的图标，可以是本地文件也可以是网络图片，当然如果暂时没有也可以不填。
                    • heroText就是主站图标下面那一行加粗加大的字，一般填写网站名称。
                    • tagline就是大字下面那一行小字，也是随意填写。
