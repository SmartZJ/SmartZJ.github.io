                      # vuepress博客搭建4使用theme-hope主题(续)

                      上一篇文章修改的都是第一眼看到的东西，这一章我们就来讲第二眼看到的东西。

                      # 内容区说明

                      网页向下滑动来到主页内容区，内容区大致分为四个部：1项目列表、2文章列表、3博主信息、4分类与标签。

                      # 项目列表配置

                      项目列表是在主页README.md中配置的它在文件里长这样：

                      ---
projects:
  - icon: project （选填）
    name: 项目名称（必填）
    desc: 项目详细描述（选填）
    link: https://你的项目链接（必填）
---

                      demo基本已经写的很详细了，这里只需要特别说一下icon这个参数，它可以设置icon也可以设置图片，不论是本地图片还是网络图片都可以。

                      # 文章列表

                      所有文章都会默认被添加到文章列表中渲染在 /article/ 路径下，在没有特别设置的情况下会按照文章时间的先后顺序排列，如果你希望在文章列表中置顶特定文章，只需在文章的 frontmatter 中将 sticky 设置为 true。

                      如果你不希望该列表包含一些特定的文章，只需在文章的 frontmatter 中将 article 设置为 false。

                      # 博主信息

                      博主信息是在theme.ts文件中配置

                      export default hopeTheme({
    ......
    blog: {
    avatar:"这里可以设置头像，如果不设置默认使用站点logo",
    name:"设置名字"
    description: "设置介绍",
    intro: "/intro.html", //个人介绍页面，点击头像可跳转到该页面
    medias:{//社交媒体链接
        Email:"https://example.com",
        .......
    	}
	}
});

                      参数我基本在代码块中做了解释，只有medias要单独提一下。

                      medias用于配置社交媒体链接：

                      • 如果社交媒体已在下方列表中，你可以直接设置 社交媒体名称: 社交媒体地址
                      • 否则，你应该传入一个元组 社交媒体名称: [社交媒体地址, 社交媒体 SVG 图标字符串或路径]

                      # 分类与标签

                      这个区域提供了四个主题自带功能：文章、分类、标签、时间轴。

                      文章小功能第一行统计了文章总数，下方的区域则用来展示收藏的文章。

                      在文章的Frontmatter 配置信息中添加star属性来标记收藏文章。

                      主题会自动收集文章的category和tag属性用来做对应的分类。

                      时间轴会根据文章的date属性来生成时间轴，没有date属性的文章默认不展示，设置了date但不想展示的文章可通过设置 timeline为false来隐藏。

                      功能相关Frontmatter 属性

                      star

                      • 类型: boolean | number
                      • 默认值: false

                      是否收藏在博客主题的文章列表中。当填入数字时，数字越大，排名越靠前。

                      category

                      • 类型: string | string[]
                      • 必填: 否

                      分类。

                      tag

                      • 类型: string | string[]
                      • 必填: 否

                      标签。

                      date

                      • 类型: DateString
                      • 必填: 否
                      • 格式: YYYY-MM-DD 或 YYYY-MM-DD hh:mm:ss

                      写作时间。

                      timeline

                      • 类型: boolean
                      • 默认值: true

                      是否将该文章添加至时间线中。

                      # 侧边栏配置

                      进入任意一篇文章，可以看到hope主题有两个侧边栏，左边的是需要配置的侧边栏，右边的是自动生成的文章大纲，这样的设计确实要比默认主题要好很多。

                      以下是sidebar.ts内容

                      import { sidebar } from "vuepress-theme-hope";

export default sidebar({
  "/": [
    "",
    {
      text: "如何使用",
      icon: "laptop-code",
      prefix: "demo/",
      link: "demo/",
      children: "structure",
    },
    {
      text: "文章",
      icon: "book",
      prefix: "posts/",
      children: "structure",
    },
    "intro",
    "slides",
  ],
});

                      是不是很眼熟，它看上去和默认主题的配置一模一样，所以看看之前的文章就知道怎么配置了，这里我就说一点它们之间不一样的地方。

                      hope主题增加了icon，配置icon后可以在标题前呈现图表让侧边栏更好看，对于children属性该主题也做了增强。

                      {
      text: "如何使用", //父标题
      icon: "laptop-code", //图表
      prefix: "demo/",	
      link: "demo/",	
      children: "structure", //子项
    }

                      我只说说不一样的地方，相信你也注意到了children: "structure"这一行代码，这是hope主题新增的功能，它可以通过文件结构自动生成侧边栏。

                      但是这样会带来另一个问题，侧边栏显示的文件夹名是英文，中英文混合的侧边栏怎看都感觉不雅观，但是直接用中文文件夹名吧又有一点邪教，所以这里我们就需要使用vuepress的特性让侧边栏优雅起来。

                      vuepress在解析侧边栏路径的时候，如果文件夹下有README.md那么侧边栏就会默认使用README.md文件中的title来作为侧边栏文件夹名的显示。

                      这样解析的文件夹名就变成中文了，但是又一个新的问题出现了，README.md文件也展示到了侧边栏里面，想到以后每个分类里都有一个这样的文件心里实在是不舒服。还好官方文档给出了解决办法，当我们不希望页面被侧边栏收录时，需要在Frontmatter 中设置 index: false。

                      我们来看看给README.md加上该属性后的侧边栏：

                      到这里文件夹名的中文显示就解决了，但别高兴的太早后面还有一个坑，README.md文件主要是修改文件夹显示名称的，通常里面只会写一些文件夹里面文件的跳转目录，这样的文章相信你不会想把它展示在博文里吧，所以为了避免在博客文章列表里看到它,我们就要再给README.md的Frontmatter中设置article:false,这样每个文件夹的README.md文件就不会被博客文章列表所展示了。
